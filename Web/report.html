Report

This report documents the creation of the restaurant website and includes any functionality included and any issues encountered.

Website Design

The website was designed using the template provided in the coursework specifications. It includes a header, a navigation, an article, a sidebar and a footer. The site was developed using Notepad++.

Each section has curved corners to make a smoother look to the site. A few sections were made to stand out using opacity settings. The navigation bar uses CSS to change the colour of the buttons when hovered over.

The header, footer and body backgrounds have textures applied to them. The header and footer use the same texture. The sites logo was designed by me using Adobe Photoshop. Other images were sourced from Google Image Search.

The sites sections were sized using percentages rather than pixels to provide a fluid layout.

A separate style was added to the site to provide a print friendly layout.

XML Menu

All food and drink items were written into an XML file. These items included names, prices and descriptions. The items were then displayed using XSL.

The XSL includes a line of code that transforms the result into HTML5 rather than XHTML. The same CSS file that styles the rest of the site was used to make the page identical in appearance to the others on the site.

Xpath was used to split the XML items into starters, mains, desserts and drinks. Each are displayed in their own section on the page. Items are also ordered by price.

Booking Form

The booking form consists of several text inputs, an email input, a textarea, a select input, radio buttons and checkboxes. These are all within a table for layout purposes.

A date input could have been used but was decided against as it does not function properly within Firefox or Internet Explorer.

Basic validation is provided through the use of HTML5. This checks whether required fields are filled out and wether a valid email address has been input. A CAPTCHA is also included underneath the table to discourage use of robots.

Slideshow

Ten images were picked for the slideshow. These were placed in an array within a JavaScript file. An image was displayed using HTML. A function was written using JQuery to change the source of the images to another within the array depending on what index was given to it. The image would then fade in using animate. The function was then used to set a timer on the images for a default. Using this function, buttons designed in Adobe Photoshop were programmed to display the first, last, previous and next images.

A random image button was then added. This button used Math.Random and Math.Floor to generate a random image key and pass it to the change image function. Difficulties were found here when the random number generator produced the same number as the key for the image already being displayed. The slideshow would produce a blank slide because it couldn't animate to a new slide. This was solved by adding an if statement to the change image function where if the number generated was the same as the key it would do nothing.

Comments for each picture were then added to another array and displayed just above the pictures using z-index. A comment background was also made slightly transparent using opacity settings. These comments were animated using a similar function to the change image function and added to the functionality of the buttons.

A thumbnail bar was then added underneath the slideshow. JQuery was used to fill the bar with the images in the array and link them to the slideshow. Another function was used to move to the left and right of the list while hovering over the arrows on either end of the thumbnail bar. Within this function are checks to find the end of the array of images and stop the scroller at either end.

This slideshow was tested using a Nokia 625 Windows Phone using Internet Explorer. The slideshow functions correctly.

Mobile Application

The mobile app was created using Mobile JQuery and Phonegap. The menu was copied from the menu screen of the main website. The form was copied straight from the main website with a few alterations. The gallery instead of being a slideshow is a selection of different pages within the app. The pictures are navigated between by clicking on them.

Browser Testing

The website was tested in Google Chrome, Mozilla Firefox and Internet Explorer. Chrome and Firefox both dislayed the website correctly. A html5 shiv statement was included in the code in order for the site to display correctly in Internet Explorer.

The site was also tested using an android tablet made by Archos running Google Chrome and a Nokia 625 Windows phone running Internet Explorer. The site displayed correctly on both devices.

Speed

For speed all images have been reduced to the smallest size possible. Smaller file sizes make for faster speeds. The main sites background is a small texture image that is repeated.
